Instalasi

Ada beberapa cara untuk menginstal ArusdataJS dalam proyek Anda. Pilih metode yang paling sesuai dengan kebutuhan Anda.

Melalui NPM

Metode yang direkomendasikan untuk proyek Node.js atau aplikasi yang menggunakan build system seperti Webpack. 

npm install arusdata --save

Kemudian, Anda dapat mengimportnya dalam kode Anda: 

// ESM
import ArusdataClient from 'arusdata';

// CommonJS
const ArusdataClient = require('arusdata');

Melalui CDN

Cara tercepat untuk mulai menggunakan ArusdataJS adalah melalui CDN. Cukup tambahkan tag script ke HTML Anda: 

<script src="https://cdn.jsdelivr.net/npm/arusdata@0.1.0/dist/arusdata.min.js"></script>

Ini akan mengekspos ArusdataClient sebagai variabel global yang dapat Anda gunakan di script Anda.

Download Manual

Anda juga dapat mengunduh file library langsung dari GitHub release:

Download Latest Release

Kemudian tambahkan ke HTML Anda: 

<script src="path/to/arusdata.min.js"></script>

Konfigurasi

Setelah instalasi, Anda perlu mengkonfigurasi ArusdataJS untuk terhubung ke server Anda.

Opsi Konfigurasi

Berikut adalah opsi konfigurasi yang tersedia saat menginisialisasi ArusdataClient: 

const arusdata = new ArusdataClient({
    // URL WebSocket server (wajib)
    endpoint: 'ws://localhost:8080/arusdata',
    
    // Interval reconnect dalam milidetik (opsional, default: 3000)
    reconnectInterval: 3000,
    
    // Otomatis reconnect saat koneksi terputus (opsional, default: true)
    autoReconnect: true,
    
    // Otomatis connect saat inisialisasi (opsional, default: true)
    autoConnect: true
});

Server Configuration

Untuk menggunakan ArusdataJS, Anda perlu menjalankan server ArusdataJS. Server ini menangani koneksi WebSocket dan distribusi pesan. 

// Instalasi server
npm install arusdata-server --save

// Menjalankan server
const ArusdataServer = require('arusdata-server');
const server = new ArusdataServer({
  port: 8080,
  path: '/arusdata'
});

Secara default, server akan berjalan di port 8080 dan menyediakan endpoint WebSocket di /arusdata. Anda dapat mengubah konfigurasi ini sesuai kebutuhan.

Penggunaan Dasar

Mari kita mulai dengan contoh penggunaan dasar ArusdataJS.

Menginisialisasi Koneksi

// Inisialisasi client
const arusdata = new ArusdataClient({
  endpoint: 'ws://localhost:8080/arusdata'
});

// Menangani event koneksi berhasil
arusdata.onConnection((socketId) => {
  console.log('Connected with socket ID:', socketId);
  // Lakukan sesuatu saat koneksi berhasil
});

// Menangani event koneksi terputus
arusdata.onDisconnection(() => {
  console.log('Disconnected from server');
  // Lakukan sesuatu saat koneksi terputus
});

Catatan: Pastikan server ArusdataJS berjalan sebelum mencoba menghubungkan klien.

Memutuskan Koneksi Secara Manual

Jika perlu, Anda bisa memutuskan koneksi secara manual: 

// Memutuskan koneksi
arusdata.disconnect();

// Menghubungkan kembali
arusdata.connect();

Channels

Channels adalah konsep penting dalam ArusdataJS yang memungkinkan pengelompokan komunikasi berdasarkan topik atau kategori tertentu.

Subscribe ke Channel

// Subscribe ke channel
const channel = arusdata.subscribe('chat-room');

// Anda juga bisa menambahkan callback saat subscription berhasil
channel.onSubscription((data) => {
  console.log('Subscribed to channel with data:', data);
});

Unsubscribe dari Channel

// Unsubscribe dari channel
arusdata.unsubscribe('chat-room');

Tipe Channel

ArusdataJS mendukung beberapa tipe channel:

  • Public Channels - Dapat diakses oleh semua klien tanpa otentikasi. 
    const publicChannel = arusdata.subscribe('public-channel');
  • Private Channels - Memerlukan otentikasi, nama channel dimulai dengan "private-". 
    // Request auth token dari server
fetch('/auth', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ 
      channel_name: 'private-channel', 
      socket_id: arusdata.getSocketId() 
  })
})
.then(response => response.json())
.then(data => {
  // Subscribe ke private channel dengan token auth
  const privateChannel = arusdata.subscribe('private-channel');
  privateChannel.auth = data.auth;
});

Events

Events adalah cara berkomunikasi melalui channels. Anda dapat mengirim dan menerima events dengan data terkait.

Mengirim Events

// Cara 1: Melalui channel
const channel = arusdata.subscribe('chat-room');
channel.trigger('new-message', {
  sender: 'John',
  text: 'Hello world!',
  timestamp: new Date().toISOString()
});

// Cara 2: Melalui client
arusdata.publish('chat-room', 'new-message', {
  sender: 'John',
  text: 'Hello world!',
  timestamp: new Date().toISOString()
});

Menerima Events

// Bind ke event tertentu
channel.bind('new-message', (data) => {
  console.log(`New message from ${data.sender}: ${data.text}`);
  // Lakukan sesuatu dengan data, misalnya update UI
  displayMessage(data);
});

// Anda bisa bind ke beberapa event
channel.bind('user-typing', (data) => {
  console.log(`${data.username} is typing...`);
  showTypingIndicator(data.username);
});

Unbinding Events

// Referensi ke callback
const messageCallback = (data) => {
  console.log(`New message from ${data.sender}: ${data.text}`);
};

// Bind callback
channel.bind('new-message', messageCallback);

// Unbind callback tertentu
channel.unbind('new-message', messageCallback);

// Atau unbind semua callback untuk event tertentu
// channel.unbind('new-message');

Authentication

ArusdataJS mendukung otentikasi untuk private channels, memungkinkan Anda mengontrol akses ke channels tertentu.

Server-side Authentication

Pada sisi server, Anda perlu mengimplementasikan endpoint untuk menghasilkan token otentikasi: 

// Contoh menggunakan Express
app.post('/auth', express.json(), (req, res) => {
  const { channel_name, socket_id } = req.body;
  
  if (!channel_name || !socket_id) {
      return res.status(400).json({ error: 'Missing required parameters' });
  }
  
  // Validasi apakah user memiliki akses ke channel
  if (!userHasAccessToChannel(req.user, channel_name)) {
      return res.status(403).json({ error: 'Access denied' });
  }
  
  // Hasilkan signature untuk channel privat
  const stringToSign = `${socket_id}:${channel_name}`;
  const signature = crypto
      .createHmac('sha256', config.authSecret)
      .update(stringToSign)
      .digest('hex');
  
  res.status(200).json({
      auth: signature
  });
});

Client-side Authentication

Pada sisi client, Anda perlu mendapatkan token otentikasi sebelum berlangganan private channel: 

// Fungsi helper untuk otentikasi channel
async function subscribeToPrivateChannel(channelName) {
  try {
      // Dapatkan socket ID
      const socketId = arusdata.getSocketId();
      
      // Request auth token dari server
      const response = await fetch('/auth', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({ 
              channel_name: channelName, 
              socket_id: socketId 
          })
      });
      
      const data = await response.json();
      
      if (!data.auth) {
          throw new Error('Authentication failed');
      }
      
      // Subscribe ke private channel dengan token auth
      const channel = arusdata.subscribe(channelName);
      channel.auth = data.auth;
      
      return channel;
  } catch (error) {
      console.error('Error subscribing to private channel:', error);
      throw error;
  }
}

// Penggunaan
subscribeToPrivateChannel('private-chat')
  .then(channel => {
      console.log('Subscribed to private channel');
      // Bind ke events
      channel.bind('message', handleMessage);
  })
  .catch(error => {
      console.error('Error:', error);
  });

Penanganan Error

Penting untuk menangani error dengan benar dalam aplikasi real-time untuk memastikan user experience yang baik.

Tipe Error Umum

  • Connection Errors - Terjadi saat gagal terhubung ke server
  • Authentication Errors - Terjadi saat otentikasi private channel gagal
  • Subscription Errors - Terjadi saat gagal berlangganan channel
  • Publish Errors - Terjadi saat gagal mempublikasikan pesan

Strategi Penanganan Error

// Menggunakan try-catch untuk penanganan error
try {
  const channel = arusdata.subscribe('chat');
  
  // Coba publish pesan
  const success = channel.trigger('message', { text: 'Hello' });
  
  if (!success) {
      // Handle publish gagal
      console.error('Failed to publish message');
  }
} catch (error) {
  console.error('Error:', error);
  // Tampilkan pesan error ke user
  showErrorToUser('Gagal terhubung ke server chat');
}

// Error handling untuk koneksi
    arusdata.onDisconnection(() => {
        console.log('Disconnected from server');
        
        // Update UI untuk menampilkan status disconnected
        updateConnectionStatus('disconnected');
        
        // Tampilkan pesan ke user
        showNotification('Koneksi terputus. Mencoba menghubungkan kembali...');
    });

Praktik Terbaik

Berikut beberapa praktik terbaik saat menggunakan ArusdataJS.

Manajemen Koneksi yang Efisien

Buat satu instance ArusdataClient dan gunakan di seluruh aplikasi Anda. Menginisialisasi beberapa koneksi dapat menyebabkan overhead dan masalah performa.

Membersihkan Event Listeners

Jangan lupa untuk menggunakan unbind() untuk menghapus event listeners yang tidak lagi dibutuhkan, terutama saat komponen atau halaman dihancurkan.

Gunakan Caching untuk Data

Untuk data yang jarang berubah, pertimbangkan untuk cache data dan gunakan update real-time hanya untuk perubahan.

Optimalkan Ukuran Data

Kirim hanya data yang diperlukan dalam event. Hindari mengirim seluruh objek besar jika hanya sebagian kecil yang berubah.

Perhatikan Keamanan

Selalu gunakan private channels untuk data sensitif. Validasi data di server sebelum broadcast ke klien lain.
